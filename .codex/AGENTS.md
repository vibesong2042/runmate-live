# Running Mate Project Rules

- 이 프로젝트는 Expo/React Native 모바일 앱과 API 서버로 구성된다.
- 수정 전 전체 흐름을 먼저 확인한다.
- 바로 앞의 오류만 고치지 말고 관련 화면, API 계약, 테스트, 빌드 영향까지 같이 본다.
- 기능 추가 후 typecheck, build, test, Android bundle 검증을 우선 수행한다.
- UI 대규모 리디자인은 MVP 기능 안정화 이후 진행한다.
- 위치정보, 친구관계, 러닝 기록은 개인정보로 취급한다.
- 사용자가 비개발자임을 전제로, 원인과 조치는 쉬운 한국어로 설명한다.
- 외부 서비스 연결, 배포, 저장소 공개, 개인정보 처리 방식 변경은 먼저 확인한다.
- Superpowers가 설치된 환경에서는 균형형으로 적용한다. 인증, 위치, 개인정보, DB/API, 데이터 저장, 매칭, 알림, 배포, 사용자-visible 동작 변경은 구현 전 짧은 설계와 테스트 계획을 먼저 제시한다.
- 문구, 색상, 아이콘, README, 코드 설명, 빠른 조사 같은 작은 작업에는 과한 Superpowers 절차를 강제하지 않는다.
- 완료 전에는 만족한 요구사항, 자동 테스트, 필요한 수동 검증, 실제 의도 상태 확인 여부를 보고한다.
- 상세 기준은 `docs/SUPERPOWERS_ADOPTION.md`를 따른다.
